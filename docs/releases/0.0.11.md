# SumUp Terminal for WooCommerce 0.0.11

## Payment completion fix

- Reconciles pending reader payments against SumUp's authoritative Transactions API using the checkout's client transaction ID.
- Recognizes successful, failed, and cancelled payments even when the asynchronous reader callback is delayed or not received by WordPress.
- Prevents a completed card charge from remaining pending until the checkout interface times out and requests cancellation.

## Compatibility notes

- Reader-screen status remains available as live cashier feedback while the transaction record determines the payment result.
- WooCommerce Status logging and the optional checkout log panel continue to work as before.
- The minimum supported PHP version remains PHP 7.4.
